# Vibe Coding

**Vibe Coding** is a software development paradigm that emerged in the early 2020s and reached maturity in 2026. It prioritizes high-level intent, aesthetic "vibes," and agentic orchestration over manual, line-by-line code writing.

## Origins and Evolution
The term was originally coined by **Andrej Karpathy** (OpenAI, Tesla) to describe a workflow where a developer writes prompts while listening to music ("vibing"), accepting the AI's output with minimal manual revision.

- **Initial Perception**: Initially viewed as a pejorative term for "lazy" or "low-quality" code generation by non-programmers.
- **2026 Professional Shift**: By 2026, with the release of [[Artificial_Intelligence/Claude_Code.md|Claude Code]], **Antigravity**, and **GPT 5.4**, Vibe Coding evolved into a high-efficiency strategy for senior engineers. It shifted from "generating snippets" to "orchestrating agentic loops."

## Core Principles
Vibe Coding is not just about prompts; it is about the **elimination of friction** in the implementation layer.

1. **Agentic Orchestration**: The developer acts as a "tech lead" or "architect," defining the structure and constraints (The Harness) while delegated agents execute the implementation.
2. **Intent over Implementation**: Focus on *what* the system should do rather than *how* the specific bits are laid out.
3. **The "Vibe" as Feedback Loop**: Using the visual and functional "feel" of an application as the primary signal for iteration. If the "vibe" is off (e.g., a UI component is clunky), the developer prompts for an adjustment of the underlying "skill" or "pattern" rather than fixing a CSS line.

## Vibe Coding vs. No-Code/Low-Code
While No-Code relies on rigid, pre-built components, Vibe Coding uses LLMs to generate **flexible, custom code bases**.
- **No-Code**: Bound by the platform's abstractions.
- **Vibe Coding**: Bound only by the LLM's reasoning and the developer's ability to provide technical context ([[Artificial_Intelligence/LLM_Harness_and_Reasoning.md|Harness Integrity]]).

## The Vibe Coder's Skillset
A professional Vibe Coder in 2026 requires more, not less, technical depth than a traditional coder:
- **Architectural Vision**: Knowing how components should interact to guide the agent correctly.
- **Technical Pedantry**: The ability to describe issues in precise technical terms (e.g., "increase the kerning" instead of "it looks tight").
- **Verification Rigor**: Since the code is generated, the developer must be an expert at setting up automated test suites and benchmarks to ensure the "vibe" translates to production stability.
